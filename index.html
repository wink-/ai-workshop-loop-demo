The loop

From intent to published artifact

Loop engineering means designing the system that prompts, checks, records, and repeats. This pilot runs the loop once, manually, before any cron or unattended autonomy.

  1. 01

    Cockpit plans

    Hermes default profile cleaned up the task: create a public demo repo, make the workflow visible, verify it, and publish it.

  2. 02

    Researcher frames

    The researcher profile summarized loop engineering into discovery, framing, planning, execution, and verification.

  3. 03

    Coder implements

    claude-zai produced the implementation plan. Hermes converted it into a static HTML/CSS site with traceable artifacts.

  4. 04

    Reviewer checks

    Gemini reviews the final diff and reports correctness, accessibility, and deployment risks before publication.

  5. 05

    GitHub Ops publishes

    gh creates and configures the repository. Copilot contributes publication guidance. GitHub Actions deploys Pages.

v3 supervisor

Script-only board supervision

V3 keeps the board moving without turning the cockpit into a chat-driven event loop. A small Python script polls, diffs, and dispatches the Kanban board with deterministic subprocess calls.

Trigger

Poll first, dispatch only when allowed

The supervisor snapshots the board, reports only new changes or attention items, and can skip dispatch entirely in dry-run mode.

Read v3 supervisor doc

State

State stays outside the repo

By default the JSON state file lives under ~/.hermes/state/ai-workshop-loop-demo/, keeping the worktree clean and the last snapshot durable.

Limits

No shell=True, no foreground server

All Kanban calls go through explicit subprocess argv lists. Validation stays script-based so the loop does not depend on a running HTTP server.

v4 3D bouncing ball

Interactive physics in a tiny 3D stage.

The demo runs entirely from static assets: pause and resume the simulation, reset the sphere, and tune gravity, speed, and trail length while the ball bounces around a wireframe volume.

Space toggles play R resets Live

The ball is rendered with perspective, shadows, and a fading trail so the motion reads like a small glass box scene instead of a flat circle.

Why this matters
